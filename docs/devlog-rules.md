## 使用说明

1. **每日更新**：每天工作结束前更新当天的开发日志
2. **及时记录**：遇到重要问题或决策时及时记录，不要拖到下班
3. **详略得当**：重要内容详细记录，常规工作简要描述
4. **保持真实**：如实记录工作进展和遇到的问题
5. **定期回顾**：每周回顾开发日志，总结经验教训
6. **团队共享**：适当与团队分享有价值的技术点和踩坑经验

## 模板定制

- 可根据个人习惯调整模板结构
- 可删除不常用的章节
- 可添加项目特有的记录项
- 建议保持基本格式统一，便于团队协作和项目管理
