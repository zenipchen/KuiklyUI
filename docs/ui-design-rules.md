# Kuikly UI 设计规范 - 需求文档转设计稿

## 1. 设计流程

### 1.1 设计阶段
```
需求分析 → 信息架构 → 交互设计 → 视觉设计 → 设计交付 → 开发跟进
```

### 1.2 设计原则
- **一致性**: 保持 Kuikly 整体设计风格统一
- **简洁性**: 界面简洁清晰，减少视觉噪音
- **高效性**: 减少用户操作步骤，提高效率
- **美观性**: 现代化的视觉设计，提升品牌形象
- **跨平台**: 考虑不同平台的设计规范和特点

## 2. 设计规范

### 2.1 颜色规范

**主色调**:
- Primary: #007AFF (品牌主色)
- Secondary: #5856D6 (辅助色)

**功能色**:
- Success: #34C759 (成功)
- Warning: #FF9500 (警告)
- Error: #FF3B30 (错误)
- Info: #5AC8FA (提示)

**中性色**:
- Text Primary: #000000 / #FFFFFF
- Text Secondary: #3C3C43 / #EBEBF5
- Background: #FFFFFF / #000000
- Border: #C6C6C8 / #38383A

### 2.2 字体规范

**字号**:
- Display: 28pt / 34pt (大标题)
- Title1: 24pt / 28pt (一级标题)
- Title2: 20pt / 22pt (二级标题)
- Title3: 18pt / 20pt (三级标题)
- Body: 16pt / 17pt (正文)
- Callout: 15pt / 16pt (说明文字)
- Caption: 12pt / 13pt (辅助信息)

**字重**:
- Bold: 700 (强调)
- Medium: 500 (次要强调)
- Regular: 400 (常规)
- Light: 300 (弱化)

### 2.3 间距规范

**基础间距**: 8dp (设计系统基础单位)

**常用间距**:
- Tiny: 4dp (极小间距)
- Small: 8dp (小间距)
- Medium: 16dp (中等间距)
- Large: 24dp (大间距)
- XLarge: 32dp (超大间距)

**组件内边距**:
- Button: 12dp vertical, 16dp horizontal
- Input: 12dp all sides
- Card: 16dp all sides

### 2.4 圆角规范

- Small: 4dp (小元素)
- Medium: 8dp (按钮、输入框)
- Large: 12dp (卡片)
- XLarge: 16dp (大卡片)
- Round: 50% (圆形头像、图标)

### 2.5 阴影规范

**卡片阴影**:
- Light: 0 2dp 4dp rgba(0,0,0,0.08)
- Medium: 0 4dp 8dp rgba(0,0,0,0.12)
- Heavy: 0 8dp 16dp rgba(0,0,0,0.16)

## 3. 组件设计

### 3.1 按钮 (Button)

**主要按钮**:
- 高度: 44dp (iOS) / 48dp (Android)
- 圆角: 8dp
- 字号: 16pt
- 最小宽度: 88dp

**次要按钮**:
- 样式: 描边或透明背景
- 其他同主要按钮

**文字按钮**:
- 无背景，文字颜色为主色

**按钮状态**:
- Normal: 默认状态
- Pressed: 按下状态 (透明度 0.7)
- Disabled: 禁用状态 (透明度 0.3)

### 3.2 输入框 (Input)

**基础属性**:
- 高度: 44dp (iOS) / 48dp (Android)
- 圆角: 8dp
- 边框: 1dp solid #C6C6C8
- 内边距: 12dp
- 字号: 16pt

**状态**:
- Default: 灰色边框
- Focus: 主色边框
- Error: 错误色边框
- Disabled: 灰色背景，禁用状态

**提示文字**:
- Placeholder: #999999

### 3.3 卡片 (Card)

**基础属性**:
- 圆角: 12dp
- 阴影: 0 2dp 8dp rgba(0,0,0,0.08)
- 内边距: 16dp
- 背景: #FFFFFF

### 3.4 列表 (List)

**列表项**:
- 高度: 最小 44dp
- 分割线: 0.5dp solid #E5E5EA
- 内边距: 16dp horizontal, 12dp vertical

### 3.5 弹窗 (Dialog)

**基础属性**:
- 最大宽度: 80% 屏幕宽度
- 圆角: 16dp
- 内边距: 24dp
- 背景蒙层: rgba(0,0,0,0.5)

## 4. 平台适配

### 4.1 Android 适配

**Material Design**:
- 遵循 Material Design 3 规范
- 使用 Material 组件样式
- 支持 Android 特有的交互模式

**适配要点**:
- 状态栏、导航栏适配
- 返回键处理
- 长按、拖拽等手势

### 4.2 iOS 适配

**Human Interface Guidelines**:
- 遵循 iOS 设计规范
- 使用 iOS 原生组件样式
- 支持 iOS 特有的手势

**适配要点**:
- 安全区域适配
- 刘海屏、灵动岛适配
- 侧滑返回手势

### 4.3 鸿蒙适配

**设计特点**:
- 遵循鸿蒙设计规范
- 万能卡片设计
- 分布式场景考虑

### 4.4 Web 适配

**响应式设计**:
- 断点: 320px, 768px, 1024px, 1440px
- 流式布局
- 触摸和鼠标交互兼顾

## 5. 设计交付

### 5.1 设计稿规范

**文件命名**:
```
项目名_模块名_页面名_版本号_日期.fig
例如: Kuikly_Demo_HomePage_v1.0_20250930.fig
```

**画板组织**:
- Cover (封面)
- Overview (概览)
- Flows (流程)
- Pages (页面)
- Components (组件)
- Spec (标注)

### 5.2 标注规范

**必须标注**:
- 间距和边距
- 字号和字重
- 颜色值 (HEX)
- 圆角值
- 阴影参数
- 特殊状态说明

### 5.3 切图规范

**Android**:
- @1x, @2x, @3x, @4x
- 格式: PNG / WebP
- 命名: ic_xxx.png

**iOS**:
- @1x, @2x, @3x
- 格式: PNG / PDF (矢量图)
- 命名: ic_xxx.png

**Web**:
- @1x, @2x
- 格式: PNG / SVG
- 命名: icon-xxx.png

### 5.4 组件库

**维护组件库**:
- 基础组件
- 业务组件
- 图标库
- 色彩库

**更新流程**:
- 新增组件需评审
- 变更组件需通知开发
- 定期清理废弃组件

## 7. 开发协作

### 7.1 设计交接

**验收标准**:
- 视觉还原度 > 95%

**验收流程**:
1. 开发自测
2. 设计师验收
3. 问题反馈修改
4. 最终确认通过

## 9. 无障碍设计

### 9.1 色彩对比度

**WCAG 2.1 标准**:
- 正常文字: 对比度 ≥ 4.5:1
- 大文字 (18pt+): 对比度 ≥ 3:1
- UI 组件: 对比度 ≥ 3:1

### 9.2 可访问性

- 所有交互元素可触达
- 提供文字替代方案
- 支持屏幕阅读器
- 支持键盘导航
- 合理的焦点顺序

## 10. 性能优化

### 10.1 图片优化

- 使用合适的图片格式
- 压缩图片大小
- 使用矢量图（图标）
- 懒加载图片

### 10.2 动效优化

- 动效时长 200-300ms
- 使用硬件加速
- 避免复杂动画
- 保持 60fps
